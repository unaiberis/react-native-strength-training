# Team Model Refactoring — PocketBase Collections

> Field definitions for recreating team-related collections in PocketBase Admin UI.
> Generated: 2026-07-09

---

## 1. Collection: `teams`

**Type:** Base

| Field | Type | Rules |
|-------|------|-------|
| `name` | Text (plain) | Required |
| `description` | Text (plain) | Optional |
| `created_by` | Relation (`users`) | Required — admin who created the team |

**API Rules (suggested):**

| Rule | Setting |
|------|---------|
| List/Search | `@request.auth.id != ""` (authenticated users) |
| View | `@request.auth.id != ""` |
| Create | `@request.auth.role = "coach" \|\| @request.auth.role = "admin"` |
| Update | `created_by = @request.auth.id` (creator only) |
| Delete | `created_by = @request.auth.id` |

---

## 2. Collection: `team_memberships`

**Type:** Base

| Field | Type | Rules |
|-------|------|-------|
| `user_id` | Relation (`users`) | Required |
| `team_id` | Relation (`teams`) | Required |
| `role` | Select (`admin`, `coach`, `athlete`) | Required, max 1 value |
| `position` | Text (plain) | Optional — e.g. "Head Coach", "Assistant" |
| `joined_at` | Autogenerated (created) | PocketBase sets this automatically |

**API Rules (suggested):**

| Rule | Setting |
|------|---------|
| List/Search | `@request.auth.id != ""` |
| View | `@request.auth.id != ""` |
| Create | `@request.auth.role = "coach" \|\| @request.auth.role = "admin"` |
| Update | `@request.auth.role = "admin"` (admin only) |
| Delete | `@request.auth.role = "admin"` |

**Indexes:** Create composite unique index on `user_id + team_id` to prevent duplicate memberships.

---

## 3. Collection: `team_invites`

**Type:** Base

| Field | Type | Rules |
|-------|------|-------|
| `team_id` | Relation (`teams`) | Required |
| `code` | Text (plain) | Required, unique — 8-char alphanumeric |
| `role` | Select (`admin`, `coach`, `athlete`) | Required |
| `max_uses` | Number (integer) | Optional, nullable (null = unlimited) |
| `used_count` | Number (integer) | Default: 0 |
| `expires_at` | Date (ISO) | Optional, nullable (null = no expiry) |
| `created_by` | Relation (`users`) | Required — who generated the invite |

**API Rules (suggested):**

| Rule | Setting |
|------|---------|
| List/Search | `@request.auth.id != ""` |
| View | `@request.auth.id != ""` |
| Create | `@request.auth.role = "coach" \|\| @request.auth.role = "admin"` |
| Update | `created_by = @request.auth.id` |
| Delete | `created_by = @request.auth.id` |

**Indexes:** Unique index on `code`.

---

## 4. Collection: `program_assignments` (modified)

**Existing fields kept, with one addition:**

| Field | Type | Notes |
|-------|------|-------|
| `team_id` | **NEW** — Relation (`teams`) | Optional, nullable. Links assignment to a specific team. |

**API Rules (suggested update):**

Add to Create/Update rules: `team_id` must reference a team where the user has `coach` or `admin` role.

## Migration Steps (Admin UI)

1. **Create `teams`** collection with fields above
2. **Create `team_memberships`** collection with fields above  
3. **Create `team_invites`** collection with fields above
4. **Add `team_id`** field to existing `program_assignments` collection
5. **Set API rules** as suggested above
6. **Run `node scripts/seed-teams.mjs --clean`** to populate test data
7. **Run `node scripts/seed-demo-data.mjs`** for workout templates + sessions

## Post-Migration Cleanup

When ready (after confirming no feature depends on `users.coach`):
1. Remove `coach` field from `users` collection in Admin UI
2. Update `UserRow` type in `src/types/pocketbase.ts` to remove the deprecated `coach` field
3. Update any remaining references in code
